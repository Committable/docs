Committable 协议是为促进开源社区协作的成长而构建的一个协议。它连接了作为规则的区块链和智能合约，作为协作方的开发者，以及开源作品。
我们在构建 Committable 协议的过程中持续进行内部讨论以及与参与使用的开发者进行交流，不断思索 Committable 协议应该向何方前进。

Committable 试图为开源社区中的协作提供更加有效率的协作方式。在开源社区中，开发者们通过协作的方式来完成开源作品，但是目前的协作方式存在一些问题：

- 开源作品获得的短期和长期激励不足，开源作者在没有稳定收入的情况下，很难持续投入到开源作品的维护和开发中
- 公众难以认知开源作者的能力与贡献，大部分时候只能使用贡献者参与的项目数量和知名度来衡量。至于具体贡献者在项目中具体贡献的价值难以进行衡量和感受
- 开源项目的组织者难以像企业一样管理开源项目，不存在以职位或者经济激励作为回报的管理模式，缺乏有效手段推动项目前进

现有解决方案，例如 Gitcoin 社区、SourceCred 协议，都在为开源协作提供一些解决方案。Gitcoin 着重于优化社区投票的公平性来为项目募集资金，SourceCred 会通过社交信息来度量开发过程中不同开发者的贡献。Committable 在切入这个问题的时候采用了一个不同的角度。我们认为开源软件开发协作的本质是人。

**在开源项目中进行协作代表了人的一种选择。在什么项目中进行什么样的贡献是一种品味，是开发者在象征性领域用以区分其他人的一种方式。**

Committable 试图以开发者为核心，将开发者在开发过程中通过 git 版本管理系统提交的 commit 作为协作的基本单位，通过 commit 来度量开发者的贡献，让开发者在开源社区中获得更加公平的认可，同时也可以让开发者在开源社区中获得更加公平的回报。我们构建了一个围绕着开发者身份的反应堆：

- 基于开发者提交的 commit 对其贡献使用科学的程序分析方式在可信环境中进行算法分析度量，生成 commit 的 meta report，存储在去中心化的文件系统上
- 将 commit 以及与之关联的 meta report 铸造为 NFT，使用区块链和智能合约来对这一贡献单元进行见证
- 资助者基于开发者的贡献报告进行捐赠，同时项目管理者基于开发者的贡献进行资助分配管理
- 这一系列过程（开发、资助、管理）都是公开透明的，这些痕迹也共同构成了参与者的数字身份

我们仍然在探索中，除了和技术领域中的开发者进行交流，也在和经济、心理学领域的专家进行合作。我们相信 Committable 能够以一种新的方式来帮助开源社区中的开发者，让他们的贡献得以持续。

我们会展示 Committable 协议的核心概念和功能。我们会通过一个示意图展示 Committable 协议的运作模式以及我们在此过程中使用到的一些核心技术。功能展示中我们会我们会展示 Committable 协议的完整使用过程：

- 关联钱包和进行 GitHub 授权
- 选择项目并生成 commit 的 Meta Report 以及将这些信息铸造为代表开发者贡献的 COMMIT NFT
- 选择项目的 Pull Request 并进行资助
- 基于 COMMIT NFT 关联的贡献量对关联的 Pull Request 获取的资助进行 claim
- 基于开发者的 COMMIT NFT 历史构建开发者的去中心化身份
