介绍 Committable：一种**去中心化**的身份协议，为开发人员彻底改变开源领域。

:::note 
在 Committable，我们的使命是赋予开发者将他们对开源软件（OSS）的贡献变现的能力，从而促进 OSS 生态系统的增长和可持续性。 
:::

## OSS 的演变

自 20 世纪 90 年代末以来，大多数开源软件（OSS）项目一直在努力维持自己的生存，并支持参与其中的开发人员。可持续性进程往往不一致，超过 99% 的 OSS 项目变得不活跃，而只有少数由谷歌、Facebook 和亚马逊等企业支持的项目蓬勃发展。

这种不一致性的主要原因之一是开发者没有获得公平的报酬，导致项目停滞，最终失去参与者（Calefato 等人，2022）。项目交接给新所有者的效果不佳（Khondhu 等人，2013）也进一步导致增长和长期成功的中断。

另一方面，得到科技巨头支持的 OSS 项目通过有针对性的活动（如会议、研讨会和竞赛）（Kuwata 等人，2015）获得了生态系统的支持。没有这样的支持，项目利益相关者失去了动力和参与度，导致项目不活跃。

这些挑战长期来看对整个 OSS 生态系统构成威胁。

开源开发者使用像 GitHub 和 GitLab 这样的平台为社区项目做出贡献，将他们的作品托管在公共代码库中，并展示他们的技能。尽管这些渠道在某种程度上有效地推广了他们的作品集，但开发者在雇主或其他利益相关者面前仍然面临关于其个人资料的真实性和工作有效性的质疑。这可能导致技能和专长的展示不足，尤其是在有众多贡献者的大型项目中。

Committable 旨在改变对开发者工作的验证和补偿方式，同时解决上述行业问题。

## 可信软件分析

Committable 是下一代去中心化开源协议，面向开发者及相关项目，将软件制品（如规范、制作文件、代码、测试用例）的完整生命周期提交系统化为 COMMIT 资产。

通过依赖可信软件分析技术（例如自动化和算法工作流、可计算证明的可验证性），Committable 确保了一个安全、公正且可负责的 token化过程，同时还证明了开发者的身份并展示了他们过去的贡献。

此外，Committable 以可信软件分析为核心，可以从开发者贡献的角度追踪和安全记录项目生命周期的所有阶段。这个记录存储在 COMMIT 中，稍后可以在利益相关者之间进行交易，展示项目演变的各个方面，促进无缝高效的工作交接。

Committable 的创新经济设计确保了贡献者得到适当的补偿以及所有利益相关者的激励，从而不断激发动力和参与度，实现成功的项目开发。

与数字文明大会等倡议以及领先的行业贡献者（例如牛津大学学院区块链研究所、ACSAC）合作，Committable 旨在与行业领导者建立强大且活跃的社区，确保持续开展教育、交流、参与和激励所有参与者的活动，最终维系 OSS 生态系统。

## 从 commit 的哈希值到 COMMIT

开发者可能想知道像*提交哈希*这样的基本功能如何改变开源生态系统。

然而，每个提交哈希对应的是关于单个开发者贡献的重要信息，这些信息可以在可信环境中进行分析。分析完成后，Committable 会生成一个*元提交*，一个计算证明以及收集到的所有数据的永久数字签名。

因此，感兴趣的各方可以查看元提交的摘要，而无需查看代码本身，从而对特定开发者对项目所做的任何贡献进行公平估值。开发者可以选择将其项目特定的元数据嵌入到 COMMIT NFT 中，并将其列在市场上供未来交易。

## COMMIT 经济学

COMMIT 是 Committable 的非同质化代币，通过链接到元数据来安全地存储开发者的贡献。

开发者使用 COMMIT 作为他们的个人数字身份资产，因为 Committable 将所有与开发者通过连接的钱包和关联的 GitHub 账户关联的 COMMIT（无论是否交易）关联起来。这使开发者能够自主展示他们的专业经验和技能，同时独立管理自己的身份。

Committable 通过在开源生态系统中创建一种身份系统来挖掘开发者在软件开发背景下的价值。

COMMIT 作为 Committable 的金融工具而设计，以便在开源生态系统中索取激励，具有预先规定的金融能力。

我们的核心愿景是建立一个自给自足的去中心化开源经济，所有利益相关者（如开发者、项目经理、赞助商）为项目作出有价值的回报（如所有权、收益、信誉）而作出贡献。

Committable 管理一个特定于区块链的挖矿池和一个特定于项目的基金池，这些池收集 COMMIT 交易的版税，并自动将它们分发给项目中的 COMMIT 持有者。

这个概念确保了随着更多的 COMMIT 被铸造和交易，COMMIT 持有者随着基金池的逐渐增长而获得更多的奖励。

**出售 COMMIT：赚取开发者版税**

在 Committable 生态系统的基础上，开发者铸造 COMMIT 以加强他们的身份，同时创建一个金融工具，帮助他们赚取潜在的版税。由于铸造 COMMIT 需要支付费用，因此预计开发者会铸造有价值的非琐碎 COMMIT，一旦交易就能确保经济回报。

成为 COMMIT 卖家可以确保项目公平地展示，以及为开发者卖家提供公平的报酬，保持项目中的参与度和动力。

**购买 COMMIT：索取社区奖励**

在每笔交易的另一端，买家购买 COMMIT 以获得对特定项目所做贡献的所有权，这使他们能够：

1. 赚取开发悬赏，
2. 从基金池中赚取版税，
3. 从挖矿池中获得社区奖励，以及 
4. 成为创新、颠覆性和流行社区的一部分。

成为 COMMIT 买家激活了 Committable 的金融机制，确保了公平分配激励以及在更广泛的生态系统社区中促进激励性的工作交接和项目延续。

**支持一个项目：维持生态系统的增长**

在每个生态系统的核心都是赞助商，他们向有前途的项目捐款，作为对其增长的信仰者或行业爱好者。Committable 赞助商支持开发中的关键事件（拉取请求），并在生态系统的治理中保持投票权，在捐赠时获得 Committable 治理代币。

成为 Committable 赞助商可以保证在生态系统的共识治理中拥有发言权和投票权，确保社区的参与度和动力，以及通过定制社区活动促进生态系统的增长。

## 结论

Committable 是一个用户友好、安全且透明的平台，旨在彻底改变开源行业和整个 Web3 生态系统。通过成为下一代去中心化开源协议的一部分，COMMIT 买家、卖家和赞助商可以共同塑造一个不断发展的新型经济，以适应不断发展的数字时代。Committable 通过安全且私密地分析所有对开源软件的贡献，公平且慷慨地补偿生态系统中的利益相关者。
